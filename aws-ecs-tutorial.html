Langkah 1: Jaringan & Keamanan (VPC)

Penting untuk Free Tier!

Agar tetap gratis, kita akan menempatkan ECS Task di Public Subnet. Kenapa? Karena Private Subnet membutuhkan NAT Gateway ($32+/bln) agar bisa menarik image dari internet/ECR.

1.1 Membuat VPC

Cari VPC di AWS Console -> Create VPC.

  • Pilih VPC and more.
  • Name tag: ecs-lab-vpc
  • IPv4 CIDR: 10.0.0.0/16
  • AZs: 2. Public Subnets: 2. Private Subnets: 0.
  • NAT Gateways: None. VPC Endpoints: None.
  • Klik Create VPC.

1.2 Security Groups (Benteng Keamanan)

Pilih menu Security Groups. Buat dua SG:

SG-ALB

Untuk Load Balancer

  • Inbound: HTTP (80) | Source: 0.0.0.0/0
  • Inbound: HTTPS (443) | Source: 0.0.0.0/0

SG-ECS-TASK

Untuk Container

  • Inbound: Custom TCP (3000) | Source: SG-ALB

Langkah 2: Setup ECS & ECR

2.1 Amazon ECR (Registry)

Tempat menyimpan Docker Image kita secara private.

  • Buka Elastic Container Registry -> Create repository.
  • Name: koperasi-app. Visibility: Private.
  • Klik Create. Simpan URI-nya (misal: 123.dkr.ecr...).

2.2 Application Load Balancer (ALB)

  • Target Group: Buat Target Group baru. Type: IP. Protocol: HTTP:3000. VPC: ecs-lab-vpc.
  • ALB: Create Load Balancer -> ALB. Name: ecs-lab-alb. Network: Pilih VPC & 2 Public Subnets. SG: SG-ALB. Listener: Forward to Target Group.

2.3 ECS Task & Service

  • Cluster: Create Cluster -> Infrastructure: AWS Fargate. Name: ecs-lab-cluster.
  • Task Definition: Buat baru. Family: koperasi-task. Image: Masukkan URI ECR. Port: 3000.
  • Service: Di dalam Cluster, klik Create Service.
    • Deployment type: Rolling Update.
    • Desired Tasks: 2.
    • Networking: Pilih 2 Public Subnet & SG-ECS-TASK.
    • Auto-assign Public IP: ENABLED (Krusial!).
    • Load Balancing: Pilih ALB & Target Group yang sudah dibuat.

Langkah 3: GitHub Actions CI/CD

Rolling update memungkinkan kita update aplikasi tanpa downtime. GitHub akan mendorong image baru, dan ECS akan mengganti task lama satu per satu.

CONFIG setup GitHub Secrets

Di Repo GitHub -> Settings -> Secrets -> Actions, tambahkan:

AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY

3.1 Workflow YAML (Rolling Update)

name: Deploy to Amazon ECS

on:
  push:
    branches: [ "main" ]

env:
  AWS_REGION: ap-southeast-1
  ECR_REPOSITORY: koperasi-app
  ECS_SERVICE: ecs-lab-service
  ECS_CLUSTER: ecs-lab-cluster
  ECS_TASK_DEFINITION: .aws/task-definition.json # Simpan file JSON task-def di repo
  CONTAINER_NAME: app-container

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v1
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ${{ env.AWS_REGION }}

      - name: Login to Amazon ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v1

      - name: Build, tag, and push image to ECR
        id: build-image
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG .
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG
          echo "image=$ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG" >> $GITHUB_OUTPUT

      - name: Fill in the new image ID in the ECS task definition
        id: task-def
        uses: aws-actions/amazon-ecs-render-task-definition@v1
        with:
          task-definition: ${{ env.ECS_TASK_DEFINITION }}
          container-name: ${{ env.CONTAINER_NAME }}
          image: ${{ steps.build-image.outputs.image }}

      - name: Deploy Amazon ECS task definition
        uses: aws-actions/amazon-ecs-deploy-task-definition@v1
        with:
          task-definition: ${{ steps.task-def.outputs.task-definition }}
          service: ${{ env.ECS_SERVICE }}
          cluster: ${{ env.ECS_CLUSTER }}
          wait-for-service-stability: true

Langkah 4: Testing & Verifikasi

Verifikasi Akses

  1. Buka menu EC2 -> Load Balancers.
  2. Copy DNS Name dari ALB Anda.
  3. Buka di browser: http://[DNS-ALB]
  4. Jika muncul halaman aplikasi Anda, berarti Traffic Flow (ALB -> ECS) berhasil.

Verifikasi Rolling Update

  1. Ubah teks sederhana di kode aplikasi Anda (misal versi 1.0 ke 1.1).
  2. git push origin main.
  3. Pantau tab Actions di GitHub.
  4. Di AWS ECS Console, lihat tab Deployments. Anda akan melihat Service menjalankan 4 Task sementara (2 lama, 2 baru) sebelum mematikan yang lama.
